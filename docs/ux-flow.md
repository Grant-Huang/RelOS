## 1. 设计原则与用户理念

### 1.1 核心设计理念

> **RelOS 不是取代工程师，而是给工程师一个永远在线的"最强副驾驶"。**

设计必须体现这一理念：
- 推荐结果永远是**建议**，不是**命令**
- 置信度必须**可见**，不能隐藏在黑盒里
- 工程师的确认/否定是**贡献**，而非"纠错"

### 1.2 工业场景设计约束

| 约束 | 设计应对 |
|------|---------|
| 工厂环境嘈杂，操作员注意力有限 | 关键信息大字体，层级分明，3 秒内看懂核心结论 |
| 手戴工作手套，触控精度低 | 操作按钮最小 44×44px，确认/否定用色块区分 |
| 网络不稳定（工厂内网） | 操作结果本地缓存，离线可查历史 |
| 工程师不信任 AI 黑盒 | 每条推荐必须展示"依据关系"，支持溯源 |
| 多语言（部分工厂操作员读写能力有限）| 图标辅助文字，减少纯文字依赖 |

### 1.3 用户心智模型

工程师看到告警推荐时的心理流程：

```
[看到推荐] → [看置信度] → [看依据] → [判断是否合理] → [确认或否定]
                ↑
            "这个有多准？"（工程师第一问题）
```

因此：**置信度必须在界面上最显眼的位置显示。**

## 2. 核心用户旅程

### 2.1 旅程 1：维修工程师处理告警

```
触发点：PDA 或工控屏收到告警推送

步骤 1：[查看告警概览]
  - 看到：设备名 + 告警码 + 严重程度
  - 耗时：< 5 秒

步骤 2：[查看根因推荐]
  - 看到：推荐根因（20 字以内）+ 置信度（70%）+ 引擎类型（规则/AI）
  - 感受：有参考，但我需要判断
  - 耗时：< 10 秒

步骤 3：[查看依据]（可选，高级操作）
  - 看到：支撑这个推荐的 2–3 条历史关系
  - 感受：哦，这是张工之前录进去的经验
  - 耗时：< 30 秒

步骤 4：[执行操作 + 提交反馈]
  - 操作：去检查轴承
  - 反馈：检查完确认根因正确 → 点"✓ 确认"
  - 耗时：实际维修时间后

步骤 5：[系统学习]（背后自动）
  - 系统：置信度从 0.70 升至 0.85
  - 工程师无感知，但下次同类告警会更准
```

**情感曲线**：期待（收到推送）→ 半信半疑（看到推荐）→ 理解（看到依据）→ 信任（验证正确后）→ 主动录入（开始贡献经验）

### 2.2 旅程 2：专家初始化（知识录入）

```
触发点：产品实施工程师邀请高级工程师参加"专家初始化会议"

步骤 1：[结构化访谈]
  - 实施工程师提问：这台机器最常见的故障是什么？
  - 高级工程师回答：振动告警 70% 是轴承，30% 是冷却液
  - 实施工程师实时录入系统

步骤 2：[确认录入关系]
  - 界面展示：ALARM__INDICATES__COMPONENT_FAILURE（轴承）confidence=0.70
  - 高级工程师确认：对，就是这个意思
  - 点击"确认录入"

步骤 3：[批量导入历史数据]（可选）
  - 上传 MES 导出的 Excel
  - 系统自动解析，展示预览（N 条关系）
  - 确认后批量导入

预计时长：1 小时录入 20–30 条核心关系
```

### 2.3 旅程 3：IT 管理员 API 集成

```
触发点：工厂 IT 需要将告警系统接入 RelOS

步骤 1：查阅 /docs（自动生成的 Swagger UI）
步骤 2：测试 POST /v1/decisions/analyze-alarm（用 curl 或 Postman）
步骤 3：在告警系统中集成 HTTP 调用
步骤 4：配置 Webhook 接收 HITL 触发通知（Sprint 3 实现）
预计时长：半天完成基础集成
```

## 3. 界面设计规范

### 3.1 颜色系统

| 用途 | 颜色 | Hex | 说明 |
|------|------|-----|------|
| 高置信度（≥ 0.75）| 绿色 | `#16A34A` | 规则引擎路径 |
| 中置信度（0.5–0.75）| 橙色 | `#EA580C` | LLM 路径 |
| 低置信度（< 0.5）| 红色 | `#DC2626` | HITL 路径 |
| 待审核 | 蓝色 | `#2563EB` | pending_review |
| 已归档 | 灰色 | `#6B7280` | archived |
| 主操作 | 工业蓝 | `#1D4ED8` | 主要按钮 |
| 背景 | 深灰 | `#1F2937` | 工控屏深色模式 |

### 3.2 置信度可视化组件

置信度不显示小数，显示**百分比 + 色块条**：

```
┌────────────────────────────────┐
│  根因推荐：轴承磨损             │
│                                │
│  置信度  ████████░░  70%  高可信│
│  来源    张工·20年经验           │
│  引擎    规则引擎               │
│                                │
│  [✓ 确认根因]  [✗ 不是这个]    │
└────────────────────────────────┘
```

置信度区间映射：
- 75–100%：高可信（绿色）
- 50–74%：中等（橙色）
- 0–49%：不确定（红色，HITL 触发）

### 3.3 操作按钮规范

```
主操作（确认根因）：
  背景：#16A34A（绿）| 文字：白 | 尺寸：全宽 | 字号：18px

次操作（否定/重新分析）：
  背景：#DC2626（红）| 文字：白 | 尺寸：全宽 | 字号：16px

辅助操作（查看依据）：
  背景：透明 | 边框：#6B7280 | 文字：灰 | 尺寸：全宽
```

### 3.4 信息密度原则

**一屏只讲一件事：**
- 屏幕 1：这台机器出了什么问题（告警概览）
- 屏幕 2：系统认为根因是什么（根因推荐 + 置信度）
- 屏幕 3：为什么这么认为（依据关系列表，按需展开）

### 5.1 工作台双前端 IA（MVP React）

与原型 [`relos_workbench_v2.html`](relos_workbench_v2.html) 对齐：**侧栏分组** = 用户前端·运行时 / 专家·管理员·知识训练 / 系统监控 / 分析与演示；根路径 `/` 重定向至 `/runtime/dashboard`。浅色与深色主题通过 `html.theme-light` / `html.theme-dark` 切换（`localStorage` 持久化）。

| 分组 | 视图 | 路径 | 说明 |
|------|------|------|------|
| 运行时 | 仪表盘 | `/runtime/dashboard` | 今日有事、指标折叠、一键进入告警/提示区 |
| 运行时 | 自动标注监控 | `/runtime/automation` | 埋点/规则说明；可接 `GET /v1/telemetry/events` |
| 运行时 | 提示标注（HITL） | `/runtime/prompt` | 默认仅展示置信度 0.50–0.79；`POST /v1/relations/{id}/feedback` |
| 知识训练 | 公开知识（层 1） | `/knowledge/public` | 粘贴 + 预标注占位；`knowledge_phase=bootstrap` |
| 知识训练 | 专家知识（层 2） | `/knowledge/expert` | 访谈 + 结构化向导；`?tab=interview` / `?tab=wizard` |
| 知识训练 | 企业文档（层 3） | `/knowledge/documents` | 文档列表、批审、`POST .../annotate/{rel_id}` + 提交图谱 |
| 系统 | 知识库状态 | `/system/kb-status` | `GET /v1/metrics` + 文档摘要 |
| 分析与演示 | 告警分析等 | `/alarm`、`/line-efficiency`、`/strategic-sim` | 保留原演示页 |

**兼容性重定向**：`/hitl` → `/runtime/prompt`；`/interview` → `/knowledge/expert?tab=interview`；`/expert-init` → `/knowledge/expert?tab=wizard`；`/dashboard` → `/runtime/dashboard`。

**三层权威性**：各知识训练页顶栏使用 `LayerAuthorityBar`（层编号、置信度上限、`knowledge_phase` 文案），与 [`architecture.md`](architecture.md) / 数据模型一致。

**访问入口（易混淆点）**：业务侧应引导用户打开 **API 文档 / 集成应用** 或 **`frontend` 本地开发（常见 `http://localhost:3000`）**。**`http://localhost:7474` 为 Neo4j Browser**（图数据库管理工具，需 `neo4j` 数据库账号密码），不是 RelOS 工作台。详见 [用户操作手册 §1.1](user-manual.md)。

## 6. 无障碍与工业场景适配

### 6.1 工控屏适配（800×480 分辨率）

- 最小字体：16px
- 操作按钮：高度 ≥ 60px
- 颜色对比度：AA 级以上（WCAG 2.1）

### 6.2 暗色模式（工厂低光照环境）

- 默认提供暗色主题
- 背景：`#111827`，卡片：`#1F2937`
- 减少蓝光（夜班使用）

### 6.3 离线降级

- 历史推荐结果本地缓存（IndexedDB）
- 断网时显示"当前为离线模式，显示最近缓存结果"
- 反馈操作离线队列，恢复网络后自动同步

### 6.4 响应时间反馈

| 等待时长 | 界面表现 |
|---------|---------|
| < 500ms | 无需 Loading |
| 500ms–3s | 骨架屏占位 |
| > 3s | 进度条 + "正在分析关系图谱..." |
| > 8s | 提示"AI 分析较慢，可先手动处理" |
| > 15s | 自动降级为 HITL，提示"已加入人工审核队列" |
